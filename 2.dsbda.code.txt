# cell 12 – detecting outliers using iqr method

# detecting outliers using iqr method

q1 = df['study_hours'].quantile(0.25)

q3 = df['study_hours'].quantile(0.75)

iqr = q3 - q1

lower_limit = q1 - 1.5 * iqr

upper_limit = q3 + 1.5 * iqr

print("lower limit:", lower_limit)
print("upper limit:", upper_limit)

# displaying outliers

outliers = df[
    (df['study_hours'] < lower_limit) |
    (df['study_hours'] > upper_limit)
]

print(outliers)

# theory

# what is data preprocessing?

data preprocessing is the process of cleaning and preparing raw data before analysis or machine learning.

it includes:

* handling missing values
* fixing inconsistent data
* removing outliers
* transforming variables
* encoding categorical values

---

# missing values

missing values are empty values present in dataset.

examples:

* nan
* null
* blank entries

missing values can reduce analysis accuracy and create problems during machine learning.

methods to handle missing values:

* replacing with mean
* replacing with median
* replacing with mode
* removing rows or columns

in this practical:

* missing values were replaced using mean.

---

# inconsistent values

inconsistent values are unrealistic or invalid values.

examples:

* attendance greater than 100
* marks greater than maximum possible marks

these values must be corrected before analysis.

---

# outliers

outliers are extreme values that are very different from other observations.

example:

* study hours = 15 while most students study between 2 to 6 hours

outliers can:

* affect average values
* affect machine learning models
* distort graphs

---

# iqr method for outlier detection

iqr stands for interquartile range.

formula:

IQR = Q3 - Q1

outlier limits:

Lower Limit = Q1 - 1.5 x IQR

Upper Limit = Q3 + 1.5 x IQR

values outside these limits are considered outliers.

---

# data transformation

data transformation changes data into a better format for analysis.

purposes:

* reduce skewness
* improve distribution
* improve visualization
* improve machine learning performance

---

#Skewness in data
Skewness tells whether data is distributed evenly or tilted to one side.

Positive skew (right skew): more small values, few very large values.
Negative skew (left skew): more large values, few very small values.
Zero skew: data is balanced/symmetric.

Example: salaries are often positively skewed because a few people earn very high amounts.

#Log transformation
A technique used to reduce skewness and make data more balanced by applying logarithm (log) to values.

Example:

Original: 10, 20, 30, 1000
After log: values become closer together.

Used for:

handling highly skewed data
reducing impact of large values
improving model performance

# log transformation

log transformation reduces skewness in highly skewed data.

formula:

y=log(x)

in this practical:

* log transformation was applied on study_hours column.

---

# categorical encoding

machine learning algorithms cannot understand text values directly.

therefore categorical values are converted into numbers.

methods used:

* label encoding
* one hot encoding
