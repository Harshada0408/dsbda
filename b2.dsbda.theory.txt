WHAT IS HDFS?
HDFS = Hadoop Distributed File System

Instead of storing one big file on one computer:
HDFS splits files into blocks.

Example:

10 GB file →

Block 1 → Machine A
Block 2 → Machine B
Block 3 → Machine C

Benefits:

parallel processing
fault tolerance
scalability
WHAT IS MAPREDUCE?

MapReduce is a programming model.

It has 2 main phases:

INPUT
  ↓
MAPPER
  ↓
INTERMEDIATE DATA
  ↓
REDUCER
  ↓
FINAL OUTPUT
UNDERSTANDING MAPPER

Mapper:

reads data
processes records
emits key-value pairs
YOUR PROGRAM'S MAPPER

Input log:

2015-10-18 INFO system started

Mapper output:

(INFO,1)

Meaning:

key = INFO
value = 1 occurrence
WHY VALUE = 1?

Because each line represents:
ONE occurrence.

Reducer later adds all 1s.

Example:

(INFO,1)
(INFO,1)
(INFO,1)

Reducer:

INFO : 3
UNDERSTANDING REDUCER

Reducer:

groups same keys
aggregates data
produces final result
SHUFFLE AND SORT

Very important concept.

Between Mapper and Reducer:
Hadoop automatically groups identical keys.

Example:

Mapper output:

(INFO,1)
(ERROR,1)
(INFO,1)

After shuffle/sort:

INFO → [1,1]
ERROR → [1]

Reducer then sums values.

FILE 2 — LogReducer.java
HASHMAP
Map<String, Integer> result = new HashMap<>();

Stores:

INFO → 1040
ERROR → 150
LOOP THROUGH MAPPER OUTPUT
for (String entry : mappedData)

Processes each mapper record.

SPLIT

Example:

Index	Value
0	INFO

INFO/WARN/etc.

CORE REDUCER LOGIC

        logLevel,
        result.getOrDefault(logLevel, 0) + 1

MOST IMPORTANT LINE.

WHAT DOES getOrDefault DO?

If key exists:
return value.

Else:
return 0.

Example:

New value:

RETURN RESULT

FILE 3 — LogDriver.java

for (Map.Entry<String, Integer> entry : result.entrySet())

MOST IMPORTANT VIVA QUESTIONS
1. What is MapReduce?

MapReduce is a programming model used for processing large datasets in distributed systems using Mapper and Reducer phases.

2. What does Mapper do?

Mapper reads input data and generates intermediate key-value pairs.

3. What does Reducer do?

Reducer groups identical keys and aggregates values.

4. Why use Hadoop?

Hadoop enables distributed storage and parallel processing of big data.

5. What is HDFS?

HDFS is Hadoop Distributed File System used to store large files across multiple machines.

6. Why BufferedReader?

BufferedReader efficiently reads large text files line by line.

7. Why HashMap?

HashMap provides fast key-value storage and retrieval for counting log levels.

8. What is intermediate data?

Mapper output before reducer processing is called intermediate data.

9. Why use (INFO,1)?

It represents one occurrence of INFO log level.

10. What is shuffle and sort?

Hadoop automatically groups identical keys between mapper and reducer phases.

11. Is this real Hadoop MapReduce?

No. This practical simulates the MapReduce concept using Java without actual Hadoop cluster execution.

12. Why is MapReduce scalable?

Because data processing can be divided across multiple systems and executed in parallel.

13. What happens if log.txt is missing?

FileNotFoundException occurs.

14. Why check parts.length > 2?

To avoid ArrayIndexOutOfBoundsException for malformed log lines.

15. What is fault tolerance in Hadoop?

If one node fails, replicated data blocks on other nodes prevent data loss.

16. Difference between traditional processing and MapReduce?
Traditional	MapReduce
Single machine	Multiple machines
Limited scalability	Highly scalable
Slower for huge data	Parallel processing
Less fault tolerant	Highly fault tolerant
