import org.apache.spark.sql.SparkSession

object SimplePractical {
  def main(args: Array[String]): Unit = {
    // 1. initialize spark session
    // builder helps set the app name and local master node
    val spark = SparkSession.builder()
      .appName("WordCountPractical")
      .master("local[*]")
      .getOrCreate()

    // 2. set log level to error 
    // hides the long startup information logs
    spark.sparkContext.setLogLevel("ERROR")

    // 3. create input data
    // sequence of strings to simulate a text file
    val sentences = Seq(
      "Spark is fast",
      "Scala is powerful",
      "Spark and Scala are used in Big Data",
      "Big Data is the future"
    )

    // 4. convert to rdd
    // parallelize sends the data to spark executors
    val rdd = spark.sparkContext.parallelize(sentences)

    // 5. logic: flatmap splits, map pairs, reducebykey sums
    val counts = rdd
      .flatMap(line => line.split(" "))   // breaks sentences into words
      .map(word => (word, 1))             // makes (word, 1) pairs
      .reduceByKey(_ + _)                 // adds counts for same words

    // 6. show output
    // collect brings distributed results back to the screen
    println("--- Word Count Results ---")
    counts.collect().foreach(println)

    // 7. close spark session
    spark.stop()
  }
}

# writeup

**Simple Program in Scala Using Apache Spark Framework**

---

## AIM

To write and execute a simple Scala program using the Apache Spark framework to perform distributed data processing and demonstrate basic RDD transformations and actions.

---

## THEORY

### 1. INTRODUCTION TO APACHE SPARK

Apache Spark is a distributed data processing framework used for big data analytics. It processes large datasets efficiently by distributing computation across multiple nodes (or cores in local mode).

Spark is widely used for:

* Big Data Processing
* Real-time analytics
* Machine learning pipelines
* Data transformation and ETL operations

---

### 2. SCALA AND SPARK INTEGRATION

Scala is the primary language for Apache Spark because:

* It runs on JVM
* It supports functional programming
* It integrates naturally with Spark APIs

Spark programs in Scala are typically written using:

* **SparkSession** → Entry point
* **RDD (Resilient Distributed Dataset)** → Core data structure

---

### 3. SPARK SESSION

SparkSession is the starting point of any Spark application.

It:

* Initializes Spark context
* Manages configuration
* Connects to execution environment (local or cluster)

Example:

```scala
val spark = SparkSession.builder()
  .appName("WordCountPractical")
  .master("local[*]")
  .getOrCreate()
```

---

### 4. RDD (RESILIENT DISTRIBUTED DATASET)

RDD is the fundamental data structure in Spark.

Features:

* Immutable (cannot be changed once created)
* Distributed across partitions
* Fault tolerant

RDD operations are of two types:

* **Transformations** (lazy operations)
* **Actions** (trigger execution)

---

### 5. TRANSFORMATIONS USED

#### a) flatMap

Splits each line into multiple words.

#### b) map

Transforms each word into key-value pair:
(word, 1)

#### c) reduceByKey

Aggregates values of same key to compute frequency.

---

### 6. ACTION USED

#### collect()

Brings distributed data from executors back to the driver node for display.

---

### 7. DATA PROCESSING FLOW

This program simulates a small text processing system:

Input sentences → Split into words → Map to pairs → Aggregate counts → Display result

---

### 8. ALGORITHM

1. Initialize SparkSession
2. Create input dataset (list of sentences)
3. Convert dataset into RDD
4. Split sentences into words using flatMap
5. Map each word to (word, 1)
6. Apply reduceByKey to count occurrences
7. Collect and display output
8. Stop Spark session

---

### 9. CONCEPTS USED

This practical demonstrates:

* Distributed computing
* Functional transformations
* Key-value pair processing
* RDD operations
* In-memory data processing

It simulates a basic **Word Count system**, which is a foundational Big Data problem.

---

### 10. FLOW OF EXECUTION

Input Data (Sentences)
↓
SparkSession Initialization
↓
Create RDD using parallelize()
↓
flatMap (Split words)
↓
map (Create key-value pairs)
↓
reduceByKey (Count words)
↓
collect()
↓
Display Output
